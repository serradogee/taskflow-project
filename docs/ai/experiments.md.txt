# Experimento 4: Ordenar una lista de números

## Solución sin IA

Se implementó manualmente una función para ordenar un array de números utilizando el método sort de JavaScript.

function sortNumbers(numbers){
  return numbers.sort(function(a,b){
    return a - b;
  });
}

Tiempo aproximado: 4 minutos.

## Solución con IA

Se pidió a la IA generar una función para ordenar números de menor a mayor.

function sortNumbers(numbers){
  return [...numbers].sort((a,b) => a - b);
}

Tiempo aproximado: 1 minuto.

## Comparación

La versión generada por IA utiliza el operador spread para evitar modificar el array original. Esto mejora la calidad del código y evita efectos secundarios.

# Experimento 5: Contar tareas completadas (TaskFlow)

## Solución sin IA

Se creó una función para contar cuántas tareas están completadas dentro de un array de tareas.

function countCompletedTasks(tasks){
  let count = 0;

  for(let i = 0; i < tasks.length; i++){
    if(tasks[i].completed){
      count++;
    }
  }

  return count;
}

Tiempo aproximado: 6 minutos.

## Solución con IA

Se pidió a la IA optimizar la función utilizando métodos modernos de JavaScript.

function countCompletedTasks(tasks){
  return tasks.filter(task => task.completed).length;
}

Tiempo aproximado: 2 minutos.

## Comparación

La solución con IA es más corta y utiliza métodos funcionales de JavaScript, lo que mejora la legibilidad del código.

# Experimento 6: Guardar tareas en localStorage (TaskFlow)

## Solución sin IA

Se escribió manualmente una función para guardar las tareas en el almacenamiento local del navegador.

function saveTasks(tasks){
  localStorage.setItem("tasks", JSON.stringify(tasks));
}

Tiempo aproximado: 5 minutos.

## Solución con IA

La IA generó una función con manejo básico de errores.

function saveTasks(tasks){
  try{
    localStorage.setItem("tasks", JSON.stringify(tasks));
  }catch(error){
    console.error("Error saving tasks", error);
  }
}

Tiempo aproximado: 2 minutos.

## Comparación

La versión generada con IA incluye manejo de errores, lo que hace el código más robusto y preparado para posibles fallos.

En este documento se investiga qué es el Model Context Protocol (MCP), cómo configurarlo en Cursor y cómo utilizarlo para acceder a información de un repositorio de GitHub.

# ¿Qué es Model Context Protocol?

Model Context Protocol (MCP) es un protocolo que permite conectar modelos de inteligencia artificial con herramientas externas. Gracias a MCP, la IA puede acceder a información adicional como archivos del sistema, repositorios de GitHub o bases de datos.

Esto permite que la IA tenga más contexto para analizar proyectos y ayudar en tareas de programación.

En lugar de copiar y pegar código en el chat, la IA puede acceder directamente al repositorio y analizar el proyecto completo.

---

# Configuración de MCP en Cursor

Cursor permite conectar diferentes servidores MCP para ampliar las capacidades de la inteligencia artificial.

El proceso general consiste en:

1. Instalar un servidor MCP.
2. Conectar el servidor con Cursor.
3. Autorizar el acceso al repositorio de GitHub.
4. Realizar consultas a la IA utilizando la información del repositorio.

# Utilidad de MCP en proyectos reales

El uso de MCP en proyectos reales puede ser muy útil porque permite que la inteligencia artificial tenga acceso directo al código del repositorio.

Algunos ejemplos de uso son:

- Analizar grandes proyectos automáticamente.
- Detectar errores o malas prácticas en el código.
- Generar documentación técnica del proyecto.
- Refactorizar funciones complejas.
- Ayudar a nuevos desarrolladores a entender el proyecto rápidamente.

En proyectos grandes con muchos archivos, MCP permite a la IA analizar todo el repositorio y proporcionar sugerencias más completas y precisas.
